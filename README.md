# Django Testing Project

Сборник тестов для учебных проектов **YaNote** и **YaNews**, написанных в рамках спринта по тестированию Django-приложений. Проекты покрыты модульными тестами с использованием `unittest` и `pytest`.

## Структура проекта

```
.
├── ya_note/ # Проект YaNote (заметки)
│ ├── notes/ # Основное приложение
│ │ ├── tests/ # Тесты на unittest
│ │ │ ├── test_routes.py
│ │ │ ├── test_content.py
│ │ │ └── test_logic.py
│ │ └── ...
│ └── ...
├── ya_news/ # Проект YaNews (новости)
│ ├── news/ # Основное приложение
│ │ ├── pytest_tests/ # Тесты на pytest
│ │ │ ├── conftest.py
│ │ │ ├── test_routes.py
│ │ │ ├── test_content.py
│ │ │ └── test_logic.py
│ │ └── ...
│ └── ...
├── requirements.txt # Зависимости проекта
├── run_tests.sh # Скрипт для запуска всех тестов
├── structure_test.py # Проверка структуры репозитория
└── README.md
```

## Тестирование YaNote (unittest)

Проект заметок покрыт тестами с использованием встроенного модуля `unittest`. Тесты находятся в директории `ya_note/notes/tests/`.

### test_routes.py
- Главная страница доступна анонимному пользователю
- Страница списка заметок (`notes/`), успешного добавления (`done/`) и создания заметки (`add/`) доступны только авторизованным
- Страницы отдельной заметки, её редактирования и удаления доступны только автору (иначе 404)
- Анонимный пользователь перенаправляется на логин при попытке доступа к защищённым страницам
- Страницы регистрации, входа и выхода доступны всем

### test_content.py
- Контекст страницы списка заметок содержит `object_list` с нужной заметкой
- Заметки разных пользователей не перемешиваются
- На страницы создания и редактирования заметки передаются формы

### test_logic.py
- Создавать заметки может только авторизованный пользователь
- Нельзя создать две заметки с одинаковым `slug`
- Автоматическая генерация `slug` через `pytils.translit.slugify`
- Редактировать и удалять заметку может только её автор

## Тестирование YaNews (pytest)

Проект новостей покрыт тестами с использованием фреймворка `pytest`. Тесты находятся в директории `ya_news/news/pytest_tests/`.

### test_routes.py
- Главная страница и страница отдельной новости доступны анонимам
- Страницы редактирования и удаления комментария доступны только автору
- Анонимный пользователь перенаправляется на логин при попытке редактирования/удаления
- Авторизованный пользователь не может редактировать/удалять чужие комментарии (404)
- Страницы регистрации, входа и выхода доступны анонимам

### test_content.py
- На главной странице не более 10 новостей
- Новости отсортированы от свежих к старым
- Комментарии отсортированы в хронологическом порядке (старые в начале)
- Форма комментария доступна только авторизованным пользователям

### test_logic.py
- Аноним не может отправить комментарий
- Авторизованный пользователь может создать комментарий
- Запрещённые слова блокируют публикацию комментария с ошибкой формы
- Автор может редактировать и удалять свой комментарий
- Чужой комментарий нельзя редактировать или удалять

## Запуск тестов

### Предварительные требования
- Python 3.9+
- Установленные зависимости:
```bash
pip install -r requirements.txt
```

### Запуск всех тестов
```bash
# Используя скрипт
./run_tests.sh

# Или вручную
python manage.py test ya_note/notes/tests/  # Тесты YaNote
pytest ya_news/news/pytest_tests/           # Тесты YaNews
```

## Используемые технологии
- **Django** — веб-фреймворк
- **unittest** — встроенная библиотека для тестирования
- **pytest** — расширенный фреймворк для тестирования
- **pytest-django** — плагин для тестирования Django-приложений
- **pytils** — для работы с транслитерацией
